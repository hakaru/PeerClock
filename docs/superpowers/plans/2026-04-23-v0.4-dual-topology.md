# PeerClock v0.4.0 — Dual-Topology Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `.mesh` / `.star` / `.auto` topology selection to the `PeerClock` facade as a unified API (Q3:A), preserving bit-for-bit wire compatibility with v0.2.x in `.mesh` mode (Q1:hard) so already-shipped PeerClockMetronome v1.0 devices keep syncing with v0.4.0 mesh builds.

**Architecture:**
- `PeerClock(topology: .mesh)` → existing `WiFiTransport` + `CoordinatorElection` + binary `MessageCodec`. Zero wire changes. This is the default.
- `PeerClock(topology: .star(role: .auto))` → `StarTransport` + `HostElection` + RFC 6455 WebSocket + tagged-union JSON (`ControlMessage`/`NtpMessage`). New wire format, not compatible with v0.2.x.
- `PeerClock(topology: .star(role: .clientOnly))` → never becomes host; designed for AUv3 extensions where `NWListener` lifecycle is unsafe.
- `PeerClock(topology: .auto(.peerCountThreshold(N)))` → starts in mesh, switches to star when discovered peer count ≥ N. Transition is explicit (stop old transport, start new) with a brief disconnection window.
- Mesh and star advertise on **different Bonjour service types** (`_peerclock._tcp` for mesh, `_peerclockstar._tcp` for star) so they do not attempt to interconnect. This also means v1.0 Metronome (mesh) and v0.4.0 star builds cannot see each other — by design.
- `Transport` protocol loses `send(_:to:)` (Q5:B). All unicast goes through broadcast + receive-side filter. Internal `CommandRouter` refactors accordingly.
- Apps integration is a **separate follow-up plan** in the `PeerClockApps` repo (pin Metronome to revision of v0.4.0 mesh; DAWSync/NTP to `.star(.clientOnly)` for extension safety).

**Tech Stack:** Swift 6.0 strict concurrency, Network.framework (`NWListener`/`NWConnection`), Bonjour, `os_signpost`, XCTest. iOS 17+ / macOS 14+.

**Branch:** `feat/v0.4-dual-topology` (branched from `feat/v0.3-star-transport`; this worktree: `/Users/hakaru/DEVELOP/PeerClock-v0.4`).

**Out of scope (explicitly deferred):**
- App-side integration (separate plan in PeerClockApps).
- Remote Config / Firebase kill-switch (app concern).
- Dual-stack mesh+star on the same peer simultaneously.
- Topology transitions *during* an active sync session for `.star` (only `.auto` transitions, and only at a peer-count threshold crossing).

# Topology Spec (v0.4.0)

## Modes

### `.mesh`
- Transport: `WiFiTransport` (Bonjour + Network.framework TCP/UDP)
- Election: `CoordinatorElection` (smallest PeerID wins)
- Wire: binary `MessageCodec` (version byte `0x02`)
- Bonjour service type: `_peerclock._tcp` + `_peerclock._udp`
- Bit-compatible with v0.2.x (PeerClockMetronome v1.0 syncs with v0.4.0 mesh peers).

### `.star(role:)`
- Transport: `StarTransport` (host runs `NWListener`, clients `NWConnection`, RFC 6455 WebSocket)
- Election: `HostElection` actor with term/session_generation, settle period, jitter
- Wire: tagged-union JSON `ControlMessage`/`NtpMessage` over WebSocket text frames
- Bonjour service type: `_peerclockstar._tcp`
- TXT: `role`, `peer_id`, `term`, `score`, `proto_ver=2`
- Role: `.auto` (participate in election) or `.clientOnly` (never advertise as host, skip election, browse only — for AUv3 extensions)

### `.auto(heuristic:)`
- Starts in `.mesh`.
- Observes peer count. When heuristic triggers (e.g. `peerCountThreshold(5)` → peers.count ≥ 5), transitions to `.star(role: .auto)`.
- Transition is explicit: stop mesh runtime, start star runtime. Brief (~2s) disconnection window.
- No reverse transition in this release (once star, stays star).

## Cross-mode interop

Mesh and star advertise on different Bonjour service types — they do not discover each other. Intentional: a device running `.mesh` cannot sync with a device running `.star`. This is the user-visible cost of the dual-topology design.

## Default

`PeerClock()` defaults to `.mesh` to preserve v0.2.x compatibility for existing consumers (`from: "0.x"` SPM requirements).

## Breaking changes from v0.3.0-beta.1

- `PeerClock.init(transportFactory:)` removed from public API. Topology is the new knob.
- `Transport.send(_:to:)` removed from protocol (Q5:B). All delivery goes through `broadcast(_:)`; recipient filtering is the application's concern.
```

# `.auto` Topology Heuristic

## Trigger

`AutoHeuristic.peerCountThreshold(N)` — transition from mesh → star when **distinct** Bonjour-discovered peers (including self) ≥ N for at least `settleWindow` (default 3s).

Settle window prevents flapping on short-lived discovery races.

## Transition procedure

1. Record the peer-count threshold crossing timestamp.
2. After `settleWindow`, if still above threshold, begin transition.
3. `MeshRuntime.stop()` — cancels streams, closes connections, unadvertises.
4. Emit `TopologyEvent.transitionStarted(from: .mesh, to: .star(.auto))` on a dedicated AsyncStream.
5. `StarRuntime.start()` — registers BonjourAdvertiser on `_peerclockstar._tcp`, begins HostElection.
6. Re-publish `PeerClock.peers` once election settles.

## No reverse

Once in star, stays in star for the lifetime of the `PeerClock` instance. A user who wants to return to mesh must `stop()` and create a new `PeerClock(topology: .mesh)`.

## Rationale

- Star topology shines at 5+ devices (mesh N² connections become wasteful).
- Below 5 devices, mesh avoids a single point of failure (host election + host kill).
- The threshold is configurable (`.peerCountThreshold(N)`) for future tuning.

## Future heuristics (not in v0.4.0)

- `networkType(.cellular → .mesh)` — mesh only on local Wi-Fi
- `deviceClass(.lowPower → .clientOnly)` — weak devices never host
```

# Client-Only Star Role (for AUv3 extensions)

## Motivation

iOS App Extensions (AUv3) have:
- No guaranteed `NWListener` lifetime (host process may suspend/kill the extension)
- Lower memory ceiling
- Background-mode constraints

Running `StarHost` in an AUv3 extension risks: silent listener death, port reuse conflicts, NWListener failing to rebind after host suspension. Apple has no officially supported pattern for server sockets in extensions.

## Behavior

`PeerClock(topology: .star(role: .clientOnly))`:

1. Does **not** call `HostElection.start()`.
2. Does **not** publish a Bonjour service for hosting (`role=host` never broadcast).
3. **Does** publish a Bonjour service for visibility (`role=client-only` in TXT) so hosts know this peer will not volunteer as host.
4. Uses `BonjourBrowser` only to find the current host.
5. `StarClient` connects to whichever peer advertises `role=host`.
6. If no host exists (all peers are client-only), sync never establishes — the app displays "no host available" via `PeerClock.syncState = .idle`.

## TXT contract

Client-only peers advertise `_peerclockstar._tcp` with TXT:
```
role=client-only
peer_id=<uuid>
proto_ver=2
```

Score/term are absent (client-only peers do not participate in election).

## Host-side responsibility

`HostElection` must ignore peers with `role=client-only` when computing score/quorum. They contribute to peer count but not to host candidacy.
```

import Foundation

/// Network topology selection for a `PeerClock` instance.
///
/// See `docs/spec/topology.md`.
public enum Topology: Sendable {
    case mesh
    case star(role: StarRole = .auto)
    case auto(heuristic: AutoHeuristic = .peerCountThreshold(5))

    public static func star() -> Topology { .star(role: .auto) }
    public static func auto() -> Topology { .auto(heuristic: .peerCountThreshold(5)) }
}

/// Behavior when a peer runs in `.star` mode.
public enum StarRole: Sendable, Equatable {
    /// Participate in `HostElection`; may be elected host.
    case auto
    /// Never become host. For AUv3 extensions. See `docs/spec/client-only-role.md`.
    case clientOnly
}

/// Heuristic that drives the `.auto` mode transition from mesh to star.
public enum AutoHeuristic: Sendable, Equatable {
    /// Transition when discovered peers (including self) ≥ `n` for a settle window.
    case peerCountThreshold(Int)
}
```

        election = nil

        self.role = role
        self.configuration = configuration
        self.advertiser = BonjourAdvertiser(

        var pc: AsyncStream<[Peer]>.Continuation!
        self.peerStream = AsyncStream { pc = $0 }
        self.peerContinuation = pc

        var cc: AsyncStream<(PeerID, Command)>.Continuation!
        self.commandStream = AsyncStream { cc = $0 }
        self.commandContinuation = cc
    }

    func start() async throws {
        advertiser.start()
        try browser.start()

        if role == .auto {
            let election = HostElection(
                localPeerID: localPeerID.uuid,
                transport: transport as! StarTransport,
                browser: browser,
                advertiser: advertiser,
                termStore: termStore

            for await discovered in browser.peerStream {

                    .map { Peer(id: PeerID($0.peerID), state: .connected) }
                self?.peerContinuation.yield(peers)
            }
        }
    }

    func stop() async {
        browserForwardTask?.cancel()
        if let election { await election.stop() }
        browser.stop()
        advertiser.stop()
        await transport.stop()
    }

    var currentPeerCount: Int {
        get async { 0 /* overridden by AutoRuntime */ }
    }
}

import Foundation

internal actor AutoRuntime: TopologyRuntime {
    enum Mode: Sendable, Equatable { case mesh, star }

    nonisolated let transport: any Transport  // delegates to active runtime
    nonisolated let peerStream: AsyncStream<[Peer]>
    nonisolated let commandStream: AsyncStream<(PeerID, Command)>

    private let localPeerID: PeerID
    private let heuristic: AutoHeuristic
    private let configuration: Configuration
    private let settleWindow: Duration
    private var active: (any TopologyRuntime)?
    private var mode: Mode = .mesh
    private var settleTask: Task<Void, Never>?
    private let peerContinuation: AsyncStream<[Peer]>.Continuation
    private let commandContinuation: AsyncStream<(PeerID, Command)>.Continuation

    init(
        localPeerID: PeerID,
        heuristic: AutoHeuristic,
        configuration: Configuration,
        settleWindow: Duration = .seconds(3)
    ) {
        self.localPeerID = localPeerID
        self.heuristic = heuristic
        self.configuration = configuration
        self.settleWindow = settleWindow
        self.transport = /* initially mesh transport; rewire on transition */
            WiFiTransport(localPeerID: localPeerID, configuration: configuration)
        var pc: AsyncStream<[Peer]>.Continuation!
        self.peerStream = AsyncStream { pc = $0 }
        self.peerContinuation = pc
        var cc: AsyncStream<(PeerID, Command)>.Continuation!
        self.commandStream = AsyncStream { cc = $0 }
        self.commandContinuation = cc
    }

    func start() async throws {
        let mesh = MeshRuntime(localPeerID: localPeerID, configuration: configuration)
        active = mesh
        try await mesh.start()

        Task { [weak self] in
            for await peers in mesh.peerStream {
                await self?.onPeers(peers)
                self?.peerContinuation.yield(peers)
            }
        }
    }

    func stop() async {
        settleTask?.cancel()
        await active?.stop()
    }

    nonisolated var currentPeerCount: Int {
        get async {
            await (active?.currentPeerCount) ?? 0
        }
    }

    private func onPeers(_ peers: [Peer]) async {
        guard case .peerCountThreshold(let n) = heuristic, mode == .mesh else { return }
        if peers.count >= n {
            if settleTask == nil {
                settleTask = Task { [weak self] in
                    try? await Task.sleep(for: self?.settleWindow ?? .seconds(3))
                    await self?.transitionToStar()
                }
            }
        } else {
            settleTask?.cancel()
            settleTask = nil
        }
    }

    private func transitionToStar() async {
        guard mode == .mesh else { return }
        await active?.stop()
        let star = StarRuntime(localPeerID: localPeerID, role: .auto, configuration: configuration)
        do {
            try await star.start()
            active = star
            mode = .star
            Task { [weak self] in
                for await peers in star.peerStream { self?.peerContinuation.yield(peers) }
            }
        } catch {
            // Rollback: restart mesh. Log.
        }
    }

    #if DEBUG
    var testHook_currentMode: Mode { mode }
    func testHook_injectDiscoveredPeers(count: Int) async {
        let fake = (0..<count).map { Peer(id: PeerID(UUID()), state: .connected) }
        await onPeers(fake)
    }
    #endif
}
